# Retrospective Reference
Use when: a `retro` run needs the retrospective entry contract after a merged PR or an explicitly chosen session retrospective.
Required by: `retro`; separating durable reusable lessons from one-off incident records.
Side effects: none
Do not use when: the underlying PR is not merged and no session retrospective was explicitly chosen, or the user asked for implementation or review work instead.

After merge, write a retrospective, separate durable reusable knowledge from incident records, and avoid storing PR numbers, commit SHAs, or single-session completion logs as durable memory unless generalized.
